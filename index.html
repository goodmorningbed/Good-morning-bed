Project Overview

Abstract, Problem Statement & System Goals

ℹ️
Abstract: A smart bedroom system designed to improve sleep habits by replacing jarring alarms with a natural, step-by-step wake-up routine.

🚨 The Problem

Loud alarms shock the system, raising stress hormones and causing profound grogginess. This abrupt awakening disrupts natural sleep cycles, leaving the user feeling unrested regardless of how long they slept.

🎯 The Goal

The main goal is to automate the room climate and provide a smooth wake-up using soothing sounds and natural light. Crucially, this routine is triggered only if the user is physically detected in bed, optimizing power and preventing unnecessary disturbances.

Architecture & Methodology

The Awakening Algorithm — Step-by-Step Logic Flow

🌟
The routine is meticulously designed to naturally halt the body's sleep hormones and gently wake the user, activating only when occupancy is verified.
1
Occupancy Verification

The ESP32 checks the pressure sensor; if the bed is empty, the alarm sequence stops entirely to save power.

🔍 FSR Sensor Active
2
Environmental Shift (Climate)

The Green LED turns off, simulating the AC turning off so the room gently warms up to encourage natural awakening.

🌡️ Green LED (AC Simulator) -> OFF
3
Physical Actuation (Light)

The servo motor turns 90 degrees to simulate opening the blinds for natural sunlight, effectively stopping the body's sleep hormones.

☀️ SG90 Servo -> 90°
4
Auditory Escalation (Sound)

The MP3 player starts quiet nature sounds that gradually increase in volume until the user gets out of bed.
